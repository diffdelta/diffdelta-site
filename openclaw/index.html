The Problem: Raw HTML Is a Liability

Your OpenClaw agent needs real-time intelligence — CVEs, cloud outages, release notes. The default approach? Scrape websites. But that means:

Raw Scraping

  • 43 MB+ of HTML per source, per scan
  • Prompt injection risk from untrusted content
  • Context window overflow → compaction retries
  • Stale data if you can't poll fast enough
  • Brittle scrapers that break when layouts change

DiffDelta Feeds

  • 400 bytes to check if anything changed
  • Pre-processed JSON — no scripts, no injection vectors
  • ~49K tokens max, only what actually changed
  • Change detection built in (head.json → latest.json)
  • 38+ sources maintained for you, forever

How It Works: The Heartbeat-First Protocol

DiffDelta uses a two-step polling protocol that saves your agent from wasting tokens on unchanged data. The agent learns this pattern once and uses it forever:

Step 1: Check the heartbeat (~400 bytes)

// Agent checks: "has anything changed?"
GET https://diffdelta.io/diff/head.json

// Response:
{ "cursor": "sha256:abc...", "changed": true, "ttl_sec": 60 }

Step 2: Only fetch the full feed when the cursor changed

// Cursor changed → fetch the delta
GET https://diffdelta.io/diff/latest.json

// Structured items with risk scores:
{
  "buckets": {
    "new": [
      {
        "source": "cisa_kev",
        "headline": "Zyxel Hard-Coded Credentials Vulnerability",
        "risk_score": 9.2,
        "url": "https://www.cisa.gov/..."
      }
    ]
  }
}

Result: If nothing changed, cost is ~$0.00. If something changed, your agent gets only the delta — not the entire internet.

The Safety Angle

OpenClaw v2026.2.6 added the Skill Safety Scanner to combat malicious skills and prompt injection. DiffDelta naturally extends this posture:

  • No raw HTML — DiffDelta extracts structured data at the source. Your agent never touches unprocessed web content.
  • Provenance tracking — Every item includes provenance.fetched_at and provenance.content_hash so you can verify data origin.
  • Risk scoring — Security items come pre-scored (0–10) with summary.logic explaining the rating. Your agent can triage without reading the full advisory.
  • Read-only by design — DiffDelta feeds are pure data. No executable content, no callbacks, no surprises.

Three Ways to Integrate

Option 1: MCP Server (recommended for OpenClaw)

If your OpenClaw instance supports MCP, this is zero-code:

{
  "mcpServers": {
    "diffdelta": {
      "command": "npx",
      "args": ["@diffdelta/mcp-server"]
    }
  }
}

Your agent gets 4 native tools: diffdelta_check_head, diffdelta_poll, diffdelta_poll_source, diffdelta_list_sources.

Option 2: Skill File

Download the DiffDelta skill file and add it to your OpenClaw agent's skill set. It teaches the agent the heartbeat-first protocol, available sources, and safety rules.

Option 3: Direct HTTP

Point your agent at the API directly. All endpoints are public, no auth required:

# Discover sources
curl -s https://diffdelta.io/diff/sources.json | jq '.sources[].source_id'

# Check for changes
curl -s https://diffdelta.io/diff/head.json

# Fetch the delta
curl -s https://diffdelta.io/diff/latest.json
